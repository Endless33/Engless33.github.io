Roadmap

Phase 1 — Core Architecture (Completed / In Progress)

  • X25519 handshake
  • AEAD (ChaCha20‑Poly1305) sealing
  • Session lifecycle
  • Transport abstraction
  • Minimal client prototype
  • Mutation Log established
  • Visual language established
  • Observer map established

Phase 2 — Behavioral Layer

  • Pulse mechanism
  • Rotation logic
  • Adaptive transport switching
  • Connection heuristics
  • Error‑resilience model
  • Minimal, privacy‑preserving logging framework

Phase 3 — Data Channel

  • Encrypted data tunnel
  • Multiplexing
  • Frame format
  • Packet scheduler
  • Congestion hints
  • Stateless recovery

Phase 4 — Server Architecture

  • Minimal server daemon
  • Session registry
  • Key rotation
  • Transport negotiation
  • Multi‑node awareness (future clustering)

Phase 5 — Security Hardening

  • Replay protection
  • Timing‑side‑channel mitigation
  • Key‑material lifecycle
  • Zero‑trust integration hooks

Phase 6 — Public‑Facing Layer

  • Minimal website
  • Whitepaper v1
  • Demo video
  • Controlled GitHub surface
  • Documentation portal

Phase 7 — Expansion

  • Mobile client
  • Desktop client
  • Multi‑transport engine
  • Experimental PQC handshake
  • Experimental satellite‑latency mode
